### 1.  global-commit-template.txt 파일 위치
```
c:\commit-template.txt
```

### 2. commit-template.txt 파일 내용
```
# 제목 (50자 이내) [유형] 간결하고 명확한 작업 요약
# 유형 예시 :
# - fix : 버그수정 또는 프로그램 수정
# - feat : 새로운 기능 추가
# - refactor : 코드 구조 변경(기능 변화 없이)
# - docs : 문서 관련 수정 (예 : README 변경)
# - test :테스트 관련 작업(테스트 코드 추가, 수정 등)
# - chore : 자잘한 작업 또는 빌드 설정, 패키지 업데이트 등
# - style: 코드 포멧팅, 들여스기, 공백 수정(로직 변화 없음)
# - pref : 성능개선
#
# 본문 (선택 사항, 72자 이내 각 줄) : 작업의 상세 내용과 이유
#
# 이슈 번호 (선택 사항) : # 이슈번호
# ------------------------------------------------------------
# 제목 가이드라인 : 
# - [유형]을 포함해 작업 내용을 간결하게 작성
# - 명령문 사용 ("수정함" 대신 "수정이라고 작성")
# - 대문자로 시작하며 마지막에 마침표 생샥(제목이 영문인 경우)
#
# 본문 가이드 : 
# - 무엇이, 왜 변경되었는지 설명
# - 어떻게 수정했는지 설명
# - 이슈나 기능에 대한 추가적인 설명이 필요한 경우 작성
#
# 이슈 번호 : 작업과 관련되 이슈번호가 있다면 추가해주세요.
[feat]

#이슈 번호 :
```

### 3. 커밋 템플릿 설정
```
# 파일위치 %USERPROFILE%\commit-template.txt
git config --global commit.template ~/commit-template.txt
```
